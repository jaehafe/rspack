# 从 v1 升级到 v2

本文列出从 Rspack 1.x 到 2.0 的不兼容变更。

- 查看 [Breaking changes in Rspack 2.0](https://github.com/web-infra-dev/rspack/discussions/9270) 了解更多细节
- 对于 Rsbuild 用户，查看 [Rsbuild - 从 v1 升级到 v2](https://v2.rsbuild.rs/zh/guide/upgrade/v1-to-v2)

## 使用 Agent Skills

如果你的 Coding Agent 支持 Skills，可安装 [rspack-v2-upgrade](https://github.com/rstackjs/agent-skills#rspack-v2-upgrade) 辅助迁移。

```bash
npx skills add rstackjs/agent-skills --skill rspack-v2-upgrade
```

安装后让 Agent 引导升级即可。

## 升级 Node.js

Rspack 2.0 的最低 Node.js 版本要求为 **20.19+** 或 **22.12+**，并且不再支持 Node.js 18。

## Pure ESM 包

`@rspack/core`、`@rspack/cli` 和 `@rspack/dev-server` 已改为 **pure ESM** 包，CommonJS 构建已移除。

Node.js 20+ 已原生支持通过 [require(esm)](https://nodejs.org/api/modules.html#loading-ecmascript-modules-using-require) 加载 ESM 模块，所以大多数通过 JavaScript API 使用 Rspack 的项目无需修改代码。

> 这只影响包的使用方式，不影响 Rspack 输出 CommonJS。

## 开发服务器

[@rspack/dev-server](https://npmjs.com/package/@rspack/dev-server) 2.0 主要有这些变化：

- `@rspack/cli` 不再默认依赖 `@rspack/dev-server`。使用 `rspack dev` 或 `rspack serve` 时请手动安装：

<PackageManagerTabs command="add @rspack/dev-server@beta -D" />

- 部分 [devServer](/config/dev-server) 选项有变更，包括 `devServer.proxy`、`devServer.watchFiles` 等，详见[升级文档](https://github.com/rstackjs/rspack-dev-server/blob/main/docs/migrate-v1-to-v2.md)。

同时：

- 引入了新的 [@rspack/dev-middleware](https://github.com/rstackjs/rspack-dev-middleware) 包来代替 `webpack-dev-middleware`。
- 使用 [connect-next](https://github.com/rstackjs/connect-next) 来代替 Express v4 作为默认的中间件框架。

## 产物分析

`@rspack/cli` 不再内置 `webpack-bundler-analyzer`，`--analyze` 也已移除。请改用 [Rsdoctor](/guide/optimization/use-rsdoctor)。

## 模块编译相关变更

### 修改 module.parser.javascript.exportsPresence 的默认值

[module.parser.javascript.exportsPresence](/config/module#moduleparserjavascriptexportspresence) 的默认值已从 `warn` 变为 `error`。如需保留旧行为，可显式设为 `auto`：

```js title="rspack.config.mjs"
export default {
  module: {
    parser: {
      javascript: {
        exportsPresence: 'auto',
      },
    },
  },
};
```

### 移除 module.parser.javascript.strictExportPresence

`module.parser.javascript.strictExportPresence` 已被移除，可使用 [module.parser.javascript.exportsPresence](/config/module#moduleparserjavascriptexportspresence) 控制导出内容不存在时的行为。

### 修改 module.parser.javascript.requireAlias 的默认值

[module.parser.javascript.requireAlias](/config/module#moduleparserjavascriptrequirealias) 的默认值已从 `true` 变为 `false`。如果你仍依赖重命名的 `require` 被解析并打包，可显式启用：

```js title="rspack.config.mjs"
export default {
  module: {
    parser: {
      javascript: {
        requireAlias: true,
      },
    },
  },
};
```

### 禁用 builtin:swc-loader 读取 .swcrc

`builtin:swc-loader` 不再读取 `.swcrc`。请将 SWC 配置迁移到 loader 选项中：

```js title="rspack.config.mjs"
export default {
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'builtin:swc-loader',
        options: {
          jsc: {
            parser: {
              syntax: 'ecmascript',
              jsx: true,
            },
            transform: {
              react: {
                runtime: 'automatic',
              },
            },
          },
        },
      },
    ],
  },
};
```

### 从 target 派生 loader 和 plugin 的 targets 配置

`builtin:swc-loader`、`builtin:lightningcss-loader` 和 `rspack.LightningCssMinimizerRspackPlugin` 的 `targets` 现在默认继承 [target](/config/target)。如需覆盖，可在 loader 或 plugin 选项中配置：

```js title="rspack.config.mjs"
export default {
  target: 'node',
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'builtin:swc-loader',
        options: {
          env: {
            targets: 'chrome >= 87',
          },
        },
      },
    ],
  },
};
```

## 产物相关配置变更

### 移除 output.libraryTarget

`output.libraryTarget` 已移除；改用 [output.library.type](/config/output#outputlibrarytype)。

### 移除 output.libraryExport

`output.libraryExport` 已移除；改用 [output.library.export](/config/output#outputlibraryexport)。

### 移除 output.umdNamedDefine

`output.umdNamedDefine` 已移除；改用 [output.library.umdNamedDefine](/config/output#outputlibraryumdnameddefine)。

### 移除 output.auxiliaryComment

`output.auxiliaryComment` 已移除；改用 [output.library.auxiliaryComment](/config/output#outputlibraryauxiliarycomment)。

### 修改 output.chunkLoadingGlobal 的默认值

[output.chunkLoadingGlobal](/config/output#outputchunkloadingglobal) 的默认值已从 `webpackChunk${output.uniqueName}` 变为 `rspackChunk${output.uniqueName}`。如需保留旧名称，可显式配置：

```js title="rspack.config.mjs"
export default {
  output: {
    chunkLoadingGlobal: 'webpackChunkMyApp',
  },
};
```

### 修改 output.hotUpdateGlobal 的默认值

[output.hotUpdateGlobal](/config/output#outputhotupdateglobal) 的默认值已从 `webpackHotUpdate${output.uniqueName}` 变为 `rspackHotUpdate${output.uniqueName}`。如需保留旧名称，可显式配置：

```js title="rspack.config.mjs"
export default {
  output: {
    hotUpdateGlobal: 'webpackHotUpdateMyApp',
  },
};
```

### 修改 output.trustedTypes.policyName 的默认值

[output.trustedTypes.policyName](/config/output#outputtrustedtypes) 的打底值已从 `'webpack'` 变为 `'rspack'`。只有未设置 `output.uniqueName` 时才会使用该打底值。若依赖旧值，请更新 CSP 或显式设置：

```js title="rspack.config.mjs"
export default {
  output: {
    trustedTypes: {
      policyName: 'webpack',
    },
  },
};
```

### 移除 output.charset

`output.charset` 已移除；现代浏览器中的 `<script>` 默认就是 UTF-8。

### 移除 optimization.removeAvailableModules

`optimization.removeAvailableModules` 已移除；它在 Rspack 中没有实际作用，可直接删除：

```diff title="rspack.config.mjs"
export default {
  optimization: {
-   removeAvailableModules: true,
  },
};
```

## Stats 相关变更

### 修改 stats.toJson() 的默认参数

`stats.toJson()` 在不传参时会把以下字段默认设为 `false`：

- `modules`: 模块信息
- `assets`: 资源信息
- `chunks`: chunk 信息
- `chunkGroups`: chunk group 信息
- `entryPoints`: 入口点信息

如仍需要这些信息，请显式传参：

```js
const statsData = stats.toJson({
  modules: true,
  assets: true,
  chunks: true,
  chunkGroups: true,
  entryPoints: true,
});
```

> 完整参数请参考 [Stats](/config/stats)。

### 移除 profile 和 stats.profile 配置

顶层 `profile` 和 `stats.profile` 已移除。性能分析请改用 [Rsdoctor](/guide/optimization/use-rsdoctor)：

```diff title="rspack.config.mjs"
export default {
-  profile: true,
  stats: {
-    profile: true,
  },
};
```

## 插件相关变更

### 移除 EsmLibraryPlugin

`rspack.EsmLibraryPlugin` 已移除；改用 [output.library](/config/output#outputlibrary) 生成 ESM 产物：

```diff title="rspack.config.mjs"
-import { EsmLibraryPlugin } from '@rspack/core';

export default {
-  plugins: [new EsmLibraryPlugin()],
+  output: {
+    library: {
+      type: 'modern-module',
+    },
+  },
};
```

### 移除 WarnCaseSensitiveModulesPlugin

`WarnCaseSensitiveModulesPlugin` 已移除；改用 [CaseSensitivePlugin](/plugins/webpack/case-sensitive-plugin)：

```diff title="rspack.config.mjs"
import { rspack } from '@rspack/core';

export default {
  plugins: [
-    new rspack.WarnCaseSensitiveModulesPlugin(),
+    new rspack.CaseSensitivePlugin(),
  ],
};
```

### 移除插件的 getHooks 方法

部分插件的 `getHooks` 已移除，请改用 `getCompilationHooks`：

```diff title="my-plugin.js"
class MyPlugin {
  apply(compiler) {
    compiler.hooks.compilation.tap('MyPlugin', (compilation) => {
-      const hooks = compiler.HtmlRspackPlugin.getHooks(compilation);
+      const hooks = compiler.HtmlRspackPlugin.getCompilationHooks(compilation);

      hooks.beforeEmit.tap('MyPlugin', (data, cb) => {
        // ...
      });
    });
  }
}
```

### 调整 HtmlRspackPlugin 配置项

[HtmlRspackPlugin](/plugins/rspack/html-rspack-plugin) 的 `sri` 选项已移除；改用 [SubresourceIntegrityPlugin](/plugins/rspack/subresource-integrity-plugin)。

```diff title="rspack.config.mjs"
export default {
  plugins: [
    new rspack.HtmlRspackPlugin({
-     sri: "sha256",
    }),
+   new rspack.SubresourceIntegrityPlugin({
+     hashFuncNames: ["sha256"],
+   }),
  ],
};
```

### 调整 LightningCssMinimizerRspackPlugin 配置项

[LightningCssMinimizerRspackPlugin](/plugins/rspack/lightning-css-minimizer-rspack-plugin) 移除了这些选项：

#### 移除 draft

`draft` 选项已被移除，可使用 `drafts` 配置替代。

```diff title="rspack.config.mjs"
import { LightningCssMinimizerRspackPlugin } from '@rspack/core';

export default {
  optimization: {
    minimizer: [
      new LightningCssMinimizerRspackPlugin({
-       draft: {
-         customMedia: true,
-       },
+       drafts: {
+         customMedia: true,
+       },
      }),
    ],
  },
};
```

#### 移除 cssHeadDataCompression

`cssHeadDataCompression` 选项已被移除，你可以直接移除该选项，这不会影响构建结果。

```diff title="rspack.config.mjs"
import { LightningCssMinimizerRspackPlugin } from '@rspack/core';

export default {
  optimization: {
    minimizer: [
      new LightningCssMinimizerRspackPlugin({
-       cssHeadDataCompression: true,
      }),
    ],
  },
};
```

## Resolve 行为变化

### 默认 extensions 变更

Rspack 2.0 默认用于 JavaScript 请求的 [resolve.extensions](/config/resolve#resolveextensions) 不再包含 `.wasm`。省略扩展名的请求将不再尝试 `./module.wasm`，显式导入 `.wasm` 不受影响。如需保留旧行为，可在 `resolve.byDependency` 中把 `.wasm` 加回去：

```js title="rspack.config.mjs"
export default {
  resolve: {
    byDependency: {
      esm: {
        extensions: ['.js', '.json', '.wasm'],
      },
      commonjs: {
        extensions: ['.js', '.json', '.wasm'],
      },
    },
  },
};
```

### CSS 解析 conditions

Rspack 2.0 默认的 CSS `@import` 解析 conditions 不再包含 `webpack`。如需保留旧行为，可显式加回：

```js title="rspack.config.mjs"
export default {
  resolve: {
    byDependency: {
      'css-import': {
        conditionNames: ['webpack', '...'],
      },
    },
  },
};
```

## 其他变更

### `devtool` 默认值变化

[devtool](/config/devtool) 的默认值现在取决于 `mode`：

- 在 `mode` 为 `development` 时，将 `devtool` 的默认值从 `eval` 调整为 `cheap-module-source-map`
- 在 `mode` 为 `production` 并使用 `@rspack/cli` 时，`devtool` 的默认值从 `source-map` 调整为 `false`

如仍需在生产环境生成 source map，推荐设置为 `source-map` 或 `hidden-source-map`：

```js title="rspack.config.mjs"
const isProd = process.env.NODE_ENV === 'production';

export default {
  devtool: isProd ? 'source-map' : 'cheap-module-source-map',
};
```

### 移除 Runtime module 废弃属性

这些 Runtime module 废弃属性已移除：

- `constructorName` - 构造函数名称属性
- `moduleIdentifier` - 模块标识符属性

自定义插件请改为：

```diff title="my-plugin.js"
class MyPlugin {
  apply(compiler) {
    compiler.hooks.compilation.tap('MyPlugin', (compilation) => {
      compilation.hooks.runtimeModule.tap('MyPlugin', (module) => {
-       const name = module.constructorName;
-       const id = module.moduleIdentifier;
+       // 使用其他方式获取模块信息
+       const name = module.constructor.name;
+       const id = module.identifier();
      });
    });
  }
}
```

### ProgressPlugin handler 签名变更

`ProgressPlugin` 自定义 handler 的第三个参数现在是结构化的 `info` 对象，而不是 `items: string[]`。

```diff
- (percent, msg, items) => void
+ (percent, msg, info) => void
```

`info` 包含：

- `builtModules: number`：已构建的模块数量
- `moduleIdentifier?: string`：当前构建的模块标识符（仅在 `build modules` 更新阶段提供）

### 修改 `resolve.roots` 的默认值

[resolve.roots](/zh/config/resolve#resolveroots) 的默认值已从 `[ context ]` 变为 `[]`。如果你依赖基于 `roots` 的服务器相对 URL 解析，请显式配置：

```js title="rspack.config.mjs"
export default {
  resolve: {
    roots: [context], // rspack 配置文件的 context 路径
  },
};
```

### 移除 `readResourceForScheme` hook

`NormalModule.getCompilationHooks(compilation).readResourceForScheme` 已移除，直接改用 `readResource.for(scheme)`。

### `@module-federation/runtime-tools` 声明为可选 peerDependency

`@module-federation/runtime-tools` 已从 `dependencies` 调整为可选 `peerDependencies`。如果你使用了 [ModuleFederationPlugin](/plugins/webpack/module-federation-plugin)，请显式安装它。

### 移除 `module.unsafeCache`

`module.unsafeCache` 已移除，因为它与增量构建和持久化缓存冲突。

## 实验性配置变更

- `experiments.css`：已移除；CSS 能力默认可用，使用 `module.rules[].type` 控制处理方式。
- [experiments.asyncWebAssembly](/config/experiments#experimentsasyncwebassembly)：默认值改为 `true`，因此 `.wasm` 文件默认走内置 `webassembly/async` 规则。
- `experiments.cache`：已迁移到顶层 [cache](/config/cache)。
- `experiments.rspackFuture`：已移除；`bundlerInfo` 改用 [output.bundlerInfo](/config/output#outputbundlerinfo)。
- `experiments.incremental`：已迁移到顶层 [incremental](/config/incremental)。
- `experiments.layers`：已移除；Layer 已稳定并默认启用，按规则通过 `module.rules[].layer` 开启。
- `experiments.topLevelAwait`：已移除；Top-level await 已稳定并在 ESM 中默认启用。
- `experiments.lazyCompilation`：已迁移到顶层 [lazyCompilation](/config/lazy-compilation)，`rspack.experiments.lazyCompilationMiddleware` 改为 [rspack.lazyCompilationMiddleware](/guide/features/lazy-compilation#与自定义的服务器集成)。
- `experiments.lazyBarrel`：已移除；[Lazy barrel](/guide/optimization/lazy-barrel) 默认启用。
- `experiments.typeReexportsPresence`：已移除；改用 [module.parser.javascript.typeReexportsPresence](/config/module#moduleparserjavascripttypereexportspresence)。
- `experiments.inlineConst`：已移除；改用 [optimization.inlineExports](/config/optimization#optimizationinlineexports)。
- `experiments.inlineEnum`：已移除；改用 [optimization.inlineExports](/config/optimization#optimizationinlineexports) 和 `builtin:swc-loader` 的 [collectTypeScriptInfo.exportedEnum](/guide/features/builtin-swc-loader#collecttypescriptinfoexportedenum)。
- `experiments.outputModule`：已移除；改用 [output.module](/config/output#outputmodule)。
- `experiments.parallelLoader`：已移除；通过 [module rules](/config/module-rules#rulesuseparallel) 中的 `module.rules[].use.parallel` 开启 loader 并行化。
- `builtin:swc-loader` 的 `rspackExperiments.collectTypeScriptInfo`：已移除；改用 [collectTypeScriptInfo](/guide/features/builtin-swc-loader#collecttypescriptinfo)。
- `builtin:swc-loader` 的 `rspackExperiments.import`：已迁移到顶层 [transformImport](/guide/features/builtin-swc-loader#transformimport)；旧字段在 2.0 中仍作为废弃别名保留。
- `rspack.experiments.SubresourceIntegrityPlugin`：已迁移到顶层 [SubresourceIntegrityPlugin](/plugins/rspack/subresource-integrity-plugin)。
