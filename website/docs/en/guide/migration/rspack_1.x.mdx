# Upgrading from v1 to v2

This page lists the breaking changes from Rspack 1.x to 2.0.

- For more details, see [Breaking changes in Rspack 2.0](https://github.com/web-infra-dev/rspack/discussions/9270)
- For Rsbuild users, see [Rsbuild - Upgrading from v1 to v2](https://v2.rsbuild.rs/guide/upgrade/v1-to-v2)

## Using Agent Skills

If your Coding Agent supports Skills, install [rspack-v2-upgrade](https://github.com/rstackjs/agent-skills#rspack-v2-upgrade) to help with the migration.

```bash
npx skills add rstackjs/agent-skills --skill rspack-v2-upgrade
```

Then let the agent guide the upgrade.

## Upgrade Node.js

Rspack 2.0 requires a minimum Node.js version of **20.19+** or **22.12+**, and Node.js 18 is no longer supported.

## Pure ESM packages

`@rspack/core`, `@rspack/cli`, and `@rspack/dev-server` are now **pure ESM** packages. CommonJS builds have been removed.

Node.js 20+ supports [require(esm)](https://nodejs.org/api/modules.html#loading-ecmascript-modules-using-require), so most projects using the JavaScript API do not need code changes.

> This only affects package consumption. Rspack can still build CommonJS output.

## Dev server

[@rspack/dev-server](https://npmjs.com/package/@rspack/dev-server) 2.0 includes these changes:

- `@rspack/cli` no longer depends on `@rspack/dev-server` by default. Install it manually when using `rspack dev` or `rspack serve`:

<PackageManagerTabs command="add @rspack/dev-server@beta -D" />

- Some [devServer](/config/dev-server) options changed, including `devServer.proxy` and `devServer.watchFiles`. See the [upgrade guide](https://github.com/rstackjs/rspack-dev-server/blob/main/docs/migrate-v1-to-v2.md).

It also:

- It introduces the new [@rspack/dev-middleware](https://github.com/rstackjs/rspack-dev-middleware) package to replace `webpack-dev-middleware`.
- It uses [connect-next](https://github.com/rstackjs/connect-next) instead of Express v4 as the default middleware framework.

## Bundle analysis

`@rspack/cli` no longer bundles `webpack-bundler-analyzer`, so `--analyze` has been removed. Use [Rsdoctor](/guide/optimization/use-rsdoctor) instead.

## Module compilation changes

### Changed default value of module.parser.javascript.exportsPresence

The default value of [module.parser.javascript.exportsPresence](/config/module#moduleparserjavascriptexportspresence) is now `error` instead of `warn`. Set it to `auto` to keep the old behavior:

```js title="rspack.config.mjs"
export default {
  module: {
    parser: {
      javascript: {
        exportsPresence: 'auto',
      },
    },
  },
};
```

### Removed module.parser.javascript.strictExportPresence

`module.parser.javascript.strictExportPresence` has been removed. You can use [module.parser.javascript.exportsPresence](/config/module#moduleparserjavascriptexportspresence) to control the behavior when exports don't exist.

### Changed default value of module.parser.javascript.requireAlias

The default value of [module.parser.javascript.requireAlias](/config/module#moduleparserjavascriptrequirealias) is now `false` instead of `true`. If you still rely on renamed `require` calls being parsed and bundled, enable it explicitly:

```js title="rspack.config.mjs"
export default {
  module: {
    parser: {
      javascript: {
        requireAlias: true,
      },
    },
  },
};
```

### Disabled builtin:swc-loader reading .swcrc

`builtin:swc-loader` no longer reads `.swcrc`. Move your SWC config into loader options:

```js title="rspack.config.mjs"
export default {
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'builtin:swc-loader',
        options: {
          jsc: {
            parser: {
              syntax: 'ecmascript',
              jsx: true,
            },
            transform: {
              react: {
                runtime: 'automatic',
              },
            },
          },
        },
      },
    ],
  },
};
```

### Derive loader and plugin targets from target

The `targets` configuration for `builtin:swc-loader`, `builtin:lightningcss-loader`, and `rspack.LightningCssMinimizerRspackPlugin` now defaults to [target](/config/target). Override it in loader or plugin options if needed:

```js title="rspack.config.mjs"
export default {
  target: 'node',
  module: {
    rules: [
      {
        test: /\.js$/,
        loader: 'builtin:swc-loader',
        options: {
          env: {
            targets: 'chrome >= 87',
          },
        },
      },
    ],
  },
};
```

## Output configuration changes

### Removed output.libraryTarget

`output.libraryTarget` has been removed. Use [output.library.type](/config/output#outputlibrarytype) instead.

### Removed output.libraryExport

`output.libraryExport` has been removed. Use [output.library.export](/config/output#outputlibraryexport) instead.

### Removed output.umdNamedDefine

`output.umdNamedDefine` has been removed. Use [output.library.umdNamedDefine](/config/output#outputlibraryumdnameddefine) instead.

### Removed output.auxiliaryComment

`output.auxiliaryComment` has been removed. Use [output.library.auxiliaryComment](/config/output#outputlibraryauxiliarycomment) instead.

### Changed default value of output.chunkLoadingGlobal

The default value of [output.chunkLoadingGlobal](/config/output#outputchunkloadingglobal) changed from `webpackChunk${output.uniqueName}` to `rspackChunk${output.uniqueName}`. Set it explicitly if you need the old name:

```js title="rspack.config.mjs"
export default {
  output: {
    chunkLoadingGlobal: 'webpackChunkMyApp',
  },
};
```

### Changed default value of output.hotUpdateGlobal

The default value of [output.hotUpdateGlobal](/config/output#outputhotupdateglobal) changed from `webpackHotUpdate${output.uniqueName}` to `rspackHotUpdate${output.uniqueName}`. Set it explicitly if you need the old name:

```js title="rspack.config.mjs"
export default {
  output: {
    hotUpdateGlobal: 'webpackHotUpdateMyApp',
  },
};
```

### Changed default fallback value of output.trustedTypes.policyName

The fallback value of [output.trustedTypes.policyName](/config/output#outputtrustedtypes) changed from `'webpack'` to `'rspack'`. This fallback is only used when `output.uniqueName` is unset. If you relied on the old value, update your CSP or set it explicitly:

```js title="rspack.config.mjs"
export default {
  output: {
    trustedTypes: {
      policyName: 'webpack',
    },
  },
};
```

### Removed output.charset

`output.charset` has been removed. Generated `<script>` tags already default to UTF-8 in modern browsers.

### Removed optimization.removeAvailableModules

`optimization.removeAvailableModules` has been removed. It had no effect in Rspack, so you can delete it:

```diff title="rspack.config.mjs"
export default {
  optimization: {
-   removeAvailableModules: true,
  },
};
```

## Stats changes

### Changed default parameters of stats.toJson()

`stats.toJson()` without arguments now defaults these fields to `false`:

- `modules`: Module information
- `assets`: Asset information
- `chunks`: Chunk information
- `chunkGroups`: Chunk group information
- `entryPoints`: Entry point information

Pass them explicitly if you still need them:

```js
const statsData = stats.toJson({
  modules: true,
  assets: true,
  chunks: true,
  chunkGroups: true,
  entryPoints: true,
});
```

> See [Stats](/config/stats) for the full option list.

### Removed profile and stats.profile configuration

The top-level `profile` option and `stats.profile` have been removed. Use [Rsdoctor](/guide/optimization/use-rsdoctor) for performance analysis:

```diff title="rspack.config.mjs"
export default {
-  profile: true,
  stats: {
-    profile: true,
  },
};
```

## Plugin changes

### Removed EsmLibraryPlugin

`rspack.EsmLibraryPlugin` has been removed. Configure [output.library](/config/output#outputlibrary) instead:

```diff title="rspack.config.mjs"
-import { EsmLibraryPlugin } from '@rspack/core';

export default {
-  plugins: [new EsmLibraryPlugin()],
+  output: {
+    library: {
+      type: 'modern-module',
+    },
+  },
};
```

### Removed WarnCaseSensitiveModulesPlugin

`WarnCaseSensitiveModulesPlugin` has been removed. Use [CaseSensitivePlugin](/plugins/webpack/case-sensitive-plugin) instead:

```diff title="rspack.config.mjs"
import { rspack } from '@rspack/core';

export default {
  plugins: [
-    new rspack.WarnCaseSensitiveModulesPlugin(),
+    new rspack.CaseSensitivePlugin(),
  ],
};
```

### Removed getHooks method from some plugins

Some plugin `getHooks` helpers were removed. Use `getCompilationHooks` instead:

```diff title="my-plugin.js"
class MyPlugin {
  apply(compiler) {
    compiler.hooks.compilation.tap('MyPlugin', (compilation) => {
-      const hooks = compiler.HtmlRspackPlugin.getHooks(compilation);
+      const hooks = compiler.HtmlRspackPlugin.getCompilationHooks(compilation);

      hooks.beforeEmit.tap('MyPlugin', (data, cb) => {
        // ...
      });
    });
  }
}
```

### Adjusted HtmlRspackPlugin configuration

The `sri` option on [HtmlRspackPlugin](/plugins/rspack/html-rspack-plugin) has been removed. Use [SubresourceIntegrityPlugin](/plugins/rspack/subresource-integrity-plugin) instead.

```diff title="rspack.config.mjs"
export default {
  plugins: [
    new rspack.HtmlRspackPlugin({
-      sri: "sha256",
    }),
+    new rspack.SubresourceIntegrityPlugin({
+      hashFuncNames: ["sha256"],
+    }),
  ],
};
```

### Adjusted LightningCssMinimizerRspackPlugin configuration

These [LightningCssMinimizerRspackPlugin](/plugins/rspack/lightning-css-minimizer-rspack-plugin) options were removed:

#### Removed draft

The `draft` option has been removed. Use the `drafts` configuration instead.

```diff title="rspack.config.mjs"
import { LightningCssMinimizerRspackPlugin } from '@rspack/core';

export default {
  optimization: {
    minimizer: [
      new LightningCssMinimizerRspackPlugin({
-       draft: {
-         customMedia: true,
-       },
+       drafts: {
+         customMedia: true,
+       },
      }),
    ],
  },
};
```

#### Removed cssHeadDataCompression

The `cssHeadDataCompression` option has been removed. You can remove this configuration directly without affecting the build result.

```diff title="rspack.config.mjs"
import { LightningCssMinimizerRspackPlugin } from '@rspack/core';

export default {
  optimization: {
    minimizer: [
      new LightningCssMinimizerRspackPlugin({
-       cssHeadDataCompression: true,
      }),
    ],
  },
};
```

## Resolve behavior changes

### Default extensions change

Rspack 2.0 no longer includes `.wasm` in the default [resolve.extensions](/config/resolve#resolveextensions) for JavaScript requests. Extension-less requests such as `import './module'` or `require('./module')` no longer try `./module.wasm`, while explicit `.wasm` imports still work. Add `.wasm` back to `resolve.byDependency` if you need the old behavior:

```js title="rspack.config.mjs"
export default {
  resolve: {
    byDependency: {
      esm: {
        extensions: ['.js', '.json', '.wasm'],
      },
      commonjs: {
        extensions: ['.js', '.json', '.wasm'],
      },
    },
  },
};
```

### CSS resolve conditions

Rspack 2.0 no longer includes `webpack` in the default CSS `@import` resolve conditions. Add it back explicitly if you need the old behavior:

```js title="rspack.config.mjs"
export default {
  resolve: {
    byDependency: {
      'css-import': {
        conditionNames: ['webpack', '...'],
      },
    },
  },
};
```

## Other changes

### Changed `devtool` default value

The default value of [devtool](/config/devtool) now depends on `mode`:

- When `mode` is `development`, the default value of `devtool` has been changed from `eval` to `cheap-module-source-map`
- When `mode` is `production` and you are using `@rspack/cli`, the default value of `devtool` has been changed from `source-map` to `false`

Set `devtool` to `source-map` or `hidden-source-map` if you still need production source maps:

```js title="rspack.config.mjs"
const isProd = process.env.NODE_ENV === 'production';

export default {
  devtool: isProd ? 'source-map' : 'cheap-module-source-map',
};
```

### Removed runtime module deprecated properties

These deprecated Runtime module properties were removed:

- `constructorName` - Constructor name property
- `moduleIdentifier` - Module identifier property

Update custom plugins accordingly:

```diff title="my-plugin.js"
class MyPlugin {
  apply(compiler) {
    compiler.hooks.compilation.tap('MyPlugin', (compilation) => {
      compilation.hooks.runtimeModule.tap('MyPlugin', (module) => {
-       const name = module.constructorName;
-       const id = module.moduleIdentifier;
+       // Use alternative ways to get module information
+       const name = module.constructor.name;
+       const id = module.identifier();
      });
    });
  }
}
```

### ProgressPlugin handler signature change

The third argument of a custom `ProgressPlugin` handler is now a structured `info` object instead of `items: string[]`.

```diff
- (percent, msg, items) => void
+ (percent, msg, info) => void
```

`info` includes:

- `builtModules: number`: Number of modules built so far
- `moduleIdentifier?: string`: Identifier of the module being built (only provided during `build modules` updates)

### Changed default value of `resolve.roots`

The default value of [resolve.roots](/config/resolve#resolveroots) changed from `[ context ]` to `[]`. Set it explicitly if you rely on server-relative URL resolution from `roots`:

```js title="rspack.config.mjs"
export default {
  resolve: {
    roots: [context], // the context path of the rspack config
  },
};
```

### Removed `readResourceForScheme` hook

`NormalModule.getCompilationHooks(compilation).readResourceForScheme` has been removed. Use `readResource.for(scheme)` directly.

### `@module-federation/runtime-tools` is declared as optional peerDependency

`@module-federation/runtime-tools` moved from `dependencies` to optional `peerDependencies`. If you use [ModuleFederationPlugin](/plugins/webpack/module-federation-plugin), install it explicitly.

### Removed `module.unsafeCache`

`module.unsafeCache` has been removed because it conflicts with Incremental Build and Persistent Cache.

## Experimental configuration changes

- `experiments.css`: removed; CSS is enabled by default, and CSS handling now comes from `module.rules[].type`.
- [experiments.asyncWebAssembly](/config/experiments#experimentsasyncwebassembly): now defaults to `true`, so `.wasm` files use the built-in `webassembly/async` rule unless you disable it.
- `experiments.cache`: moved to the top-level [cache](/config/cache).
- `experiments.rspackFuture`: removed; configure `bundlerInfo` with [output.bundlerInfo](/config/output#outputbundlerinfo).
- `experiments.incremental`: moved to the top-level [incremental](/config/incremental).
- `experiments.layers`: removed; layers are stable and enabled by default, and `module.rules[].layer` now enables them per rule.
- `experiments.topLevelAwait`: removed; top-level await is stable and enabled by default in ESM.
- `experiments.lazyCompilation`: moved to top-level [lazyCompilation](/config/lazy-compilation), and `rspack.experiments.lazyCompilationMiddleware` moved to [rspack.lazyCompilationMiddleware](/guide/features/lazy-compilation#integrating-with-custom-server).
- `experiments.lazyBarrel`: removed; [Lazy barrel](/guide/optimization/lazy-barrel) is enabled by default.
- `experiments.typeReexportsPresence`: removed; use [module.parser.javascript.typeReexportsPresence](/config/module#moduleparserjavascripttypereexportspresence).
- `experiments.inlineConst`: removed; use [optimization.inlineExports](/config/optimization#optimizationinlineexports).
- `experiments.inlineEnum`: removed; use [optimization.inlineExports](/config/optimization#optimizationinlineexports) plus `builtin:swc-loader`'s [collectTypeScriptInfo.exportedEnum](/guide/features/builtin-swc-loader#collecttypescriptinfoexportedenum).
- `experiments.outputModule`: removed; use [output.module](/config/output#outputmodule).
- `experiments.parallelLoader`: removed; enable loader parallelization with `module.rules[].use.parallel` in [module rules](/config/module-rules#rulesuseparallel).
- `builtin:swc-loader` `rspackExperiments.collectTypeScriptInfo`: removed; use [collectTypeScriptInfo](/guide/features/builtin-swc-loader#collecttypescriptinfo).
- `builtin:swc-loader` `rspackExperiments.import`: moved to top-level [transformImport](/guide/features/builtin-swc-loader#transformimport); the old key still works as a deprecated alias in 2.0.
- `rspack.experiments.SubresourceIntegrityPlugin`: moved to top-level [SubresourceIntegrityPlugin](/plugins/rspack/subresource-integrity-plugin).
